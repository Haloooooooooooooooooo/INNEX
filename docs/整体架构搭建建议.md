上面是GitHub参考项目，避免我们自己重复造轮子



我的项目最好不要做成“很多开源项目拼在一起”的架构，而是把它们当作**参考模块**。你自己的系统建议搭成：

> **Next.js 前端 + Supabase 数据层 + Mastra Agent 工作流 + Docling 解析服务 + RAG 问答服务 + React Flow 图谱展示**

核心原则是：
**主流程用固定 Workflow，复杂判断交给 Agent，数据统一进 Supabase。**

------

# 一、整体架构

```text
用户输入
文字 / 链接 / 图片 / 文档 / 视频链接
        ↓
收录箱 Inbox
        ↓
轻解析 Workflow
标题 / 来源 / 摘要 / 标签 / 状态
        ↓
待内化记录
        ↓
一键内化 Agent
结构化笔记 / 关键概念 / 标签 / 关联笔记
        ↓
Markdown 笔记库
        ↓
Embedding 向量化
        ↓
知识库问答 + 知识图谱
```

你的文档里已经定义了完整状态流转：**稍后看、待内化、已沉淀**，以及“一键内化”“基于此笔记提问”“知识库定位”等动作，所以架构里一定要保留这条主链路。

# 三、你的系统应该分成 6 个核心模块

## 模块 1：收录箱 Inbox

参考：**Karakeep**

负责所有内容进入系统。

```text
用户输入：
- 文字
- 链接
- 图片
- 文档
- 临时想法
- 附件组
```

收录后生成一条 `capture_item`。

状态：

```text
稍后看
待内化
已沉淀
```

这个模块不要做复杂，重点是**快**。

------

## 模块 2：轻解析 Workflow

参考：**Karakeep + Docling**

收录后立刻做轻解析。

输出：

```text
标题
来源
收录时间
状态
标签
摘要
我的理解
附件清单
```

你的文档里已经明确：不同内容类型有不同标题、来源、摘要生成规则，比如文字太短不展示摘要、视频链接内化时再读、图片超过 3 张内化时再读等。

这一层建议用固定规则，不要完全交给 Agent。

------

## 模块 3：一键内化 Agent

参考：**Mastra**

这是你项目最核心、也最适合学 Agent 的部分。

用户点击“一键内化”后：

```text
读取收录记录
↓
判断内容类型
↓
必要时调用解析工具
↓
生成结构化笔记
↓
提取标签 / 主题 / 关键概念
↓
检索相似旧笔记
↓
生成关联关系
↓
返回草稿给用户确认
↓
用户确认后保存为已沉淀笔记
```

输出一篇 Markdown 笔记：

```markdown
# 标题

## 这篇内容讲了什么

## 核心观点

## 结构化拆解

## 关键概念

## 可复用结论

## 和我已有知识的关联

## 来源依据

## 标签
```

这里一定要加 **Human-in-the-loop**：

> Agent 生成草稿，但用户确认后才入库。

------

## 模块 4：知识库问答 RAG

参考：**Vault RAG**

已沉淀笔记进入知识库后，要做：

```text
Markdown 切片
↓
生成 embedding
↓
存入 pgvector
↓
用户提问
↓
检索相关笔记片段
↓
AI 基于证据回答
↓
展示引用来源
```

你的 AI 助手不要直接“凭空答”，必须基于已沉淀笔记回答。你的文档里也要求：证据足够时展示引用来源，证据不足时明确不确定。

------

## 模块 5：知识图谱

参考：**Atomic + React Flow**

你的图谱不要一开始做重型知识图谱。

第一版做轻量图谱：

```text
主题节点
子主题节点
笔记节点
概念节点
标签节点
```

关系：

```text
主题 → 子主题
子主题 → 笔记
笔记 → 标签
笔记 → 概念
笔记 → 相似笔记
```

图谱数据存 Supabase：

```text
graph_nodes
graph_edges
```

前端用 React Flow：

```text
点击节点 → 右侧详情抽屉
悬浮节点 → 高亮相关节点
知识库定位 → 跳转并高亮目标节点
关联笔记点击 → 切换详情抽屉内容
```

------

## 模块 6：前端 AI 助手

参考：**CopilotKit**

AI 助手放在右侧抽屉 / 侧边栏。

支持两种入口：

```text
自由提问
从某条已沉淀笔记进入“基于此笔记提问”
```

回答后可以：

```text
展示引用来源
点击来源打开笔记详情
把高价值回答加入笔记
回写到“AI 助手回答”栏
```

这和你的文档里的 AI 助手设计完全对应。

# 五、Agent 设计

你可以设计 4 个 Agent。

## 1. Capture Agent

负责轻解析。

```text
输入：用户收录内容
输出：标题、来源、摘要、标签、内容类型
```

但它只辅助，不完全决定状态。

------

## 2. Internalization Agent

最核心。

```text
输入：capture_item + attachments + my_understanding
输出：结构化笔记草稿 + 标签 + 概念 + 关联笔记
```

工具：

```text
parse_document()
parse_url()
ocr_image()
transcribe_video()
search_similar_notes()
generate_note()
extract_concepts()
create_graph_edges()
```

------

## 3. Knowledge QA Agent

负责问答。

```text
输入：用户问题
动作：检索知识库
输出：带引用回答
```

工具：

```text
search_note_chunks()
get_note_detail()
rerank_sources()
generate_grounded_answer()
save_answer_to_note()
```

------

## 4. Graph Agent

负责图谱更新。

```text
输入：新笔记
输出：主题、子主题、概念、相关笔记、图谱边
```

工具：

```text
extract_topics()
extract_concepts()
find_similar_notes()
upsert_graph_node()
upsert_graph_edge()
```
