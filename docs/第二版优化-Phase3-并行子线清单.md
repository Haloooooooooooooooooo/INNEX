# Phase3-并行子线执行清单

## 目标
- 在不重建现有骨架前提下，把内化层升级为正式知识入口。

## 子线边界
- 不阻塞 Phase2 修补。
- 只做内化层相关：输入、输出结构、模型与Prompt、资产落库。

## P3-T1 输入层规范
- [x] 固定输入优先级：primary source -> supplemental source -> user_understanding。
- [x] 明确 deferred 内容在内化阶段补解析并回写来源。
- [x] 明确视频解析不作为当前前置依赖。

## P3-T2 输出结构固化
- [x] 内化正文固定 5 段：核心内容/关键概念/原文支持要点/推断与延展/关联方向。
- [x] 事实与推断显式分区，不混写。
- [x] 摘要限制为短摘要（避免冗长拼接）。

## P3-T3 正式资产落库
- [x] 正式摘要写入 notes.summary。
- [x] 正式标签写入 notes.tags。
- [x] 正式概念写入 notes.concepts。
- [x] 事实层片段进入 note_chunks（用于检索）。

## P3-T4 模型与Prompt
- [x] 决定内化模型主备策略（主模型+超时降级）。
- [x] 内化 prompt 强制“原文依据优先，推断后置”。
- [x] 标签提取与摘要提取分开 prompt，不混任务。

## P3-T5 验收门槛
- [ ] 至少 4 个样本通过：S01/S05/S10/S11。
- [x] 每条内化结果可追溯到至少 1 条 source。
- [x] 内化内容可直接被 QA 检索，不需要额外手工清洗。

## 并行推进节奏
1. Day 1：完成 T1/T2 文档与接口约束。
2. Day 2：完成 T3 落库与回归验证。
3. Day 3：完成 T4 调优与 T5 验收。
