# 第三版迭代计划（草案）

> 目标：在第二版基础上，把 QA 从“可用”推进到“稳定可规模化复用”。  
> 状态：讨论中，边聊边补充，达成共识后再冻结为执行版。

---

## 0. 当前共识（截至 2026-06-01）

1. Phase 6 已关闭（第三轮 QA 7/7 通过）。
2. QA 当前架构是单 Agent 分层：决策层 + 检索执行层 + 证据加工层 + 回答层。
3. 下一阶段重点从“回答风格修复”转向“意图识别能力与系统稳定性提升”。

## 1. 主题 A：意图识别能力升级（首要）

### A0. 升级目标

把当前“规则冷启动意图识别”升级为“可评测、可回放、可持续迭代的 QA 意图决策模块”。

升级后不只输出一个分类标签，而是输出可驱动检索与回答的结构化决策：

- `primary_intent`：主意图
- `secondary_intents`：次意图列表
- `expansion_intent`：扩展意图
- `slots`：知识场景槽位
- `confidence`：置信度
- `clarification_needed`：是否需要澄清
- `routing_policy`：检索与扩图策略建议

### A1. 意图体系与边界定义

#### 核心意图

当前保留 5 类主意图：

- `fact_query`：事实查询，解决“这是什么 / 讲了什么 / 有没有提到”
- `summary`：总结归纳，解决“帮我提炼重点 / 梳理结构”
- `comparison`：对比决策，解决“A 和 B 有什么区别 / 怎么取舍”
- `action_advice`：执行建议，解决“怎么落地 / 下一步怎么做 / 给模板”
- `retrospective`：复盘反思，解决“哪里有问题 / 怎么优化 / 有什么教训”

#### 扩展意图

保留并强化 3 类扩展意图：

- `evidence_strengthening`：用户要依据、原因、证明，图谱扩检索优先 `supports`
- `example_request`：用户要例子、模板、案例，图谱扩检索优先 `example_of`
- `related_topic_expansion`：用户要相关、延伸、类似主题，图谱扩检索优先 `related`

#### 兜底意图

新增或显式化兜底类型：

- `ambiguous`：问题目标不清，需要澄清
- `out_of_scope`：超出知识库或当前模式能力
- `small_talk`：闲聊，不进入知识库检索
- `unsafe_or_sensitive`：高风险问题，保守处理

### A2. 意图语料资产化

建立 `intent_utterance_bank`，每条样本至少包含：

- `utterance`：用户原始问法
- `primary_intent`
- `secondary_intents`
- `expansion_intent`
- `slots`
- `is_followup`
- `expected_routing`
- `confusable_with`
- `source`：真实用户 / 改写生成 / badcase 回流

每类主意图第一阶段至少准备 30 条样本：

- 10 条标准表达
- 8 条口语表达
- 5 条省略表达
- 3 条错别字或缩写
- 4 条易混淆表达

专项样本池：

- 多意图样本：如“总结一下，并给我一个可执行模板”
- 续问样本：如“继续上一个 PRD 话题，给我最小模板”
- 混淆样本：如“为什么这个方法有效”（事实查询 vs 证据增强）
- 无关联样本：如“意图识别和 PRD 有什么关联”

### A3. 识别路由升级

当前阶段不直接上重模型，先采用组合路由：

1. 规则层
- 识别强标记词：继续、上一个、模板、对比、复盘、依据、案例等。
- 输出候选意图、显式槽位、强续问信号。

2. 语义层
- 使用 embedding 或 LLM 判断处理规则难覆盖的口语、省略和混淆表达。
- 对易混意图输出候选列表和置信度。

3. LLM 决策层
- 仅在多意图、低置信度、续问冲突、扩图策略不确定时调用。
- 输出结构化 JSON，不直接写最终答案。

4. 回退层
- JSON 解析失败或置信度过低时，回退到保守规则。
- 不能因为意图识别失败而让 QA 链路无约束漂移。

### A4. 槽位抽取

槽位用于把“识别结果”变成“可执行任务参数”。

第一阶段支持以下知识场景槽位：

- `topic`：主题，如 PRD、Agent、RAG、意图识别
- `target_object`：目标对象，如模板、指标、方法、案例、模块
- `scenario`：使用场景，如面试、学习、工作、复盘
- `source_scope`：来源范围，如某条笔记、某类来源、某个标签
- `time_range`：时间范围，如最近、本周、某日期后
- `followup_anchor`：上一轮命中的 note 或主题
- `output_format`：用户想要的输出形式，如模板、清单、对比表、步骤

例子：

用户输入：“继续上一个 PRD 话题，给我一个最小模板”

预期解析：

```json
{
  "primary_intent": "action_advice",
  "secondary_intents": ["summary"],
  "expansion_intent": "example_request",
  "is_followup": true,
  "slots": {
    "topic": "PRD",
    "target_object": "最小模板",
    "output_format": "template",
    "followup_anchor": "previous_cited_note"
  },
  "confidence": 0.86,
  "clarification_needed": false
}
```

### A5. 不确定性与澄清策略

建立意图置信度分层：

- `high`：直接进入检索与回答
- `medium`：结合上下文补判；若存在高影响歧义，则追问澄清
- `low`：保守兜底，不强行扩检索

澄清策略示例：

- 多意图冲突：用户问“总结一下怎么落地”，澄清“你更想先要摘要，还是直接要执行模板？”
- 槽位缺失：用户问“给我模板”，澄清“你想基于哪条笔记或哪个主题生成模板？”
- 续问不明：用户问“继续说”，但上文有多个主题，澄清“你是想继续 PRD 话题，还是意图识别话题？”

高风险规则：

- 不让低置信度意图直接触发大范围扩图。
- 不让无关联问题强行进入 `related` 扩展。
- 不让回答层弥补检索层的明显不确定性。

### A6. 意图识别评测体系

第一阶段建立离线 benchmark，至少 80 条样本：

- 5 类主意图各 10 条：50 条
- 3 类扩展意图各 6 条：18 条
- 续问专项：8 条
- 兜底 / 无关联 / 闲聊：4 条

核心指标：

- `primary_intent_accuracy`：主意图准确率
- `secondary_intent_recall`：次意图召回率
- `expansion_intent_accuracy`：扩展意图准确率
- `slot_f1`：槽位抽取 F1
- `followup_detection_accuracy`：续问识别准确率
- `confusion_rate`：易混意图混淆率
- `clarification_precision`：该追问时是否追问
- `routing_success_rate`：意图驱动检索是否走对路径

验收建议：

- 主意图准确率 >= 85%
- 扩展意图准确率 >= 80%
- 续问识别准确率 >= 85%
- 槽位 F1 >= 75%
- 已知 badcase 关闭率 >= 90%

### A7. Badcase 回流机制

每条意图 badcase 记录：

- 用户问题
- 期望意图
- 实际意图
- 错误类型：主意图错 / 次意图漏 / 槽位漏 / 续问错 / 澄清错
- 影响链路：检索跑偏 / 扩图错误 / 回答过硬 / 引用噪音
- 修复动作
- 回归结果

优先回流案例：

- Q5：PRD 续问最小模板，历史问题是续问锚定不足
- Q7：无关联判断，历史问题是低相关引用噪音
- “为什么有效”：事实查询与证据增强混淆
- “给我模板”：缺少主题槽位时的澄清策略

## 2. 主题 B：Agent 决策层稳态化

### B1. 决策输出协议强化
- 固化决策 JSON 契约，补齐字段校验和默认回退逻辑。
- 增加解析失败降级策略，避免决策异常传导到回答层。

### B2. 续问策略精细化
- 区分：
  - 显式续问（继续/上一个）
  - 隐式续问（主题延续但无标记词）
- 对两类续问设置不同强度的锚定与扩展策略。

### B3. 停止策略参数化
- 把“够用即停”转为可调参数（证据阈值、扩展收益阈值、最大扩展步数）。
- 增加在线回放调参能力。

---

## 3. 主题 C：RAG 与证据可信性提升

### C1. 检索融合优化
- 优化内化/原文/图谱召回融合权重，减少话题漂移。
- 针对关系判断题与低证据场景继续增强 citation 去噪。

### C2. 证据质量分层
- 建立证据等级（高/中/低）与展示规则（可展示/不展示）。
- 低证据时优先“少而精”的引用策略。

### C3. 可解释性增强
- 保留系统内部判据（日志与meta），用户侧仅展示必要依据。
- 增强“为什么答/为什么不答”的可复盘能力。

---

## 4. 评测与坏例闭环（执行护栏）

### D1. Benchmark 升级
- 除通过率外新增：
  - 意图混淆率
  - 次意图遗漏率
  - 续问连续性命中率
  - 引用相关性通过率
  - 扩图有效率/错误扩图率

### D2. Badcase 工作流
- 统一 badcase 模板：现象 -> 证据 -> 归因 -> 修复 -> 回归结果。
- 每轮迭代必须有“新增 badcase + 已关闭 badcase”。

### D3. 验收规则
- 未完成回归复测，不关闭子阶段。
- 指标和案例同时达标才算通过。
