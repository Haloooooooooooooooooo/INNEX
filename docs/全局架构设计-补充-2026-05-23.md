# 全局架构设计补充（2026-05-23）

## 1. PDF 解析链路（本地实现）
- PDF 统一使用服务端 `pdfjs-dist` 直读文本，避免 `pdf.worker` 路径问题。
- 解析输出增加元信息：
  - `page_count`
  - `extracted_chars`
  - `chars_per_page`
  - `likely_scanned_pdf`（低文本密度判定）
- `parse_debug.notes` 新增：
  - `pdf_extract_meta:<file>:strategy=...,pages=...,chars=...,cpp=...`
  - `pdf_likely_scanned:<file>`

## 2. Capture 渐进回填（Phase C）
- 录入后先乐观显示记录，再短轮询回填最终解析字段（摘要/标签/notes）。
- 录入 toast 增强：若解析阶段失败，显示“已收录，摘要/标签稍后补全 + 失败原因”。

## 3. 可观测性（Phase D）
- 新增 `GET /api/parser-health`：
  - 检查 `pdfjs` 模块可导入
  - 检查 OCR 必需配置项是否存在
  - 返回解析层健康状态（与 `llm-health` 分离）

## 4. Parse Debug 结构（Phase B 部分落地）
- `parse_debug` 新增 `stages`：
  - `detect`
  - `extract`
  - `summarize`
  - `tags`

## 5. 待后续
- `raw_text_blocks` 落库（需数据库迁移字段）
- 解析阶段耗时与 trace 字段统一
