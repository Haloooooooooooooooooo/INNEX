# 第二版优化-全模块迭代计划

## 0. 迭代原则

- 先保证全链路可用，再进入模块化迭代。
- 每个模块都要有可量化指标，不做"感觉优化"。
- 先修阻断与高频问题，再做体验和性能增强。

## 1. 录入解析模块（Capture / Parse）

### 目标
- 提升文档、图片、链接录入解析成功率。
- 降低超时与无效等待。

### 迭代项
- [x] 文档类型和大小前置判断，超阈值快速返回。
- [x] PDF 多策略解析（原生提取优先，OCR 兜底）。
- [ ] URL 正文清洗（去噪、去推荐区、去模板尾巴）。
- [x] 小红书图文链接按图片数量分流（≤5 全量 OCR / >5 deferred）。
- [x] OCR 超时控制与逐模型逐尝试错误保留。
- [x] 批量录入双入口（非文档批量 + 文档批量）。

### 指标
- 解析成功率
- P95 解析时延
- 超时率 / 失败率

## 2. 内化模块（Internalize）

### 目标
- 内化结果稳定可用，结构化字段完整。
- 支持批量高效内化，减少手工操作。

### 迭代项
- [x] 质量门禁：`title/content/summary/tags/concepts` 完整性校验。
- [x] 概念兜底：`concepts` 少于阈值时自动二次抽取。
- [x] 状态持久化：关闭页面后任务继续，重进页面状态可恢复。
- [ ] 多选右键"一键内化（免确认）"：直接后台执行，不弹确认框。
- [ ] 批量执行失败可重试，成功/失败分开统计。
- [x] 内化并发化（判型 + 嵌入有界并发，`INTERNALIZE_LLM_CONCURRENCY` / `INTERNALIZE_EMBED_CONCURRENCY`）。
- [x] 重复内化改为 upsert（同源 note 复用，不再产重复节点）。

### 指标
- 内化成功率
- `concepts_non_empty_rate`
- 平均内化时长 / P95 内化时长
- 批量任务成功率

### 10.3 Badcase 分析

#### FAIL: Q5 续问断链（R7-C02 未完全修复）

**现象**：Q4 问"PRD模板包含哪些模块"→正确命中PRD笔记并回答完整。Q5 问"继续上一个PRD话题，给我一个可直接套用的最小模板"→召回了"Agent设计"笔记，完全丢失PRD上下文。

**根因分析**：
1. `isSemanticallyContinuous` 基于词面重叠判断，Q5 含"PRD"关键词，理论上应与 Q4 有重叠——但实际检索仍偏移。
2. `FOLLOWUP_REFERENCE_MARKERS` 包含"继续/上一个"，应触发续问复用——但从结果看，复用的 note 种子可能未正确传递或被新检索覆盖。
3. 核心问题：**续问复用逻辑可能只影响图谱扩检索种子，但主检索（内化+原文）仍独立走 embedding 召回**，导致"继续上一个PRD话题"被 embedding 匹配到含"Agent"的笔记（Agent设计方法论中也提到了产品思维）。

**修复方向**：
- 续问复用不仅作用于图谱扩检索种子，还应对主检索结果做 re-rank 或 filter：上轮命中的 note 在续问场景下应获得加权。
- 或者：续问场景下，将上轮命中 note 的 chunks 直接注入候选池（不依赖 embedding 重新召回）。

#### 边界观察：Q7 召回了大量不相关片段

Q7 问"意图识别和PRD有什么关联"，检索到 7 条片段（Agent设计、面试技巧、Vibe Coding 等），全部不相关。回答正确判定"暂无直接关联"——**回答层表现好，但检索层浪费了 token**。

**优化方向**：当证据分极低（<55%）且无任何片段与问题强相关时，可考虑提前截断，不将低质片段送入回答模型。

### 10.6 回答生成层问题总结（为 Phase 6 准备）

基于 7 题回答的整体观察，当前回答生成存在以下问题：

**问题 1：回答结构机械模板化**
- 所有回答都严格按"结论→依据→可执行下一步→不确定项"四段式输出。
- 用户问简单问题（如Q2"指标怎么设"）也输出完整四段，显得冗余。
- **Phase 6 对策**：6.5 回答结构骨架应作为"内在骨架"而非表面模板，简单问题可省略"可执行下一步"或合并段落。

**问题 2：回答暴露检索过程**
- 回答开头显示"意图: xxx / 证据: xxx / 证据分: xx% / 检索: K=x / 阈值=x"。
- 这是调试信息，不应出现在用户可见的回答中。
- **Phase 6 对策**：6.5 明确"不暴露检索决策过程"，调试信息仅写日志。

**问题 3：引用格式不够自然**
- 引用以"[1] 第3-7步详细描述了..."的形式出现，像论文而非对话。
- **Phase 6 对策**：6.6 "回答要像人说的话"，引用可改为"根据你的《Prompt AB测试五步法》笔记..."更自然。

**问题 4："可执行下一步"内容过于具体且重复**
- Q1/Q2/Q3 的"可执行下一步"几乎相同（都是"设计A组共情后提问、B组直接建议"），说明模型在机械复述笔记内容而非针对当前问题给出差异化建议。
- **Phase 6 对策**：6.3 回答角色应"重组，不拼贴"，下一步建议需与当前问题的具体语境绑定。

**问题 5：不确定项过度展开**
- 每个回答的"不确定项"都列出 3 条，部分内容是显而易见的（如"未明确具体阈值需自行设定"），信息密度低。
- **Phase 6 对策**：6.6 不确定性以"后置边界说明"出现，只说真正影响用户决策的不确定项，不凑数。

**问题 6：续问断链时的拒答体验差**
- Q5 直接说"当前笔记中无法提供可直接套用的最小PRD模板"，但实际上用户的 PRD 笔记里是有相关内容的（Q4 已证明）。
- 这不是回答层的问题而是检索层的问题，但回答层可以做兜底：当检测到续问场景且当前证据不足时，提示"是否需要我基于上一轮的PRD笔记继续展开"而非直接拒答。
- **Phase 6 对策**：6.3 回答角色在 partial/insufficient 场景下，应结合对话历史给出更有帮助的引导。
