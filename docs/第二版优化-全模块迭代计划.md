# 第二版优化-全模块迭代计划

## 0. 迭代原则

- 先保证全链路可用，再进入模块化迭代。
- 每个模块都要有可量化指标，不做“感觉优化”。
- 先修阻断与高频问题，再做体验和性能增强。

## 1. 录入解析模块（Capture / Parse）

### 目标
- 提升文档、图片、链接录入解析成功率。
- 降低超时与无效等待。

### 迭代项
- 文档类型和大小前置判断，超阈值快速返回。
- PDF 多策略解析（原生提取优先，OCR 兜底）。
- URL 正文清洗（去噪、去推荐区、去模板尾巴）。

### 指标
- 解析成功率
- P95 解析时延
- 超时率 / 失败率

## 2. 内化模块（Internalize）

### 目标
- 内化结果稳定可用，结构化字段完整。
- 支持批量高效内化，减少手工操作。

### 迭代项
- 质量门禁：`title/content/summary/tags/concepts` 完整性校验。
- 概念兜底：`concepts` 少于阈值时自动二次抽取。
- 状态持久化：关闭页面后任务继续，重进页面状态可恢复。
- 多选右键“一键内化（免确认）”：直接后台执行，不弹确认框。
- 批量执行失败可重试，成功/失败分开统计。

### 指标
- 内化成功率
- `concepts_non_empty_rate`
- 平均内化时长 / P95 内化时长
- 批量任务成功率

## 3. 知识图谱模块（Graph）

### 目标
- 提升关系准确性与可解释性，减少误连漏连。

### 迭代项
- 关系类型判定规则优化（supports/related/contradicts）。
- 证据字段规范化（来源片段、置信度、触发规则）。
- 图渲染稳定性修复（避免空实例调用导致报错）。

### 指标
- 关系准确率
- 漏连率 / 误连率
- 图页面错误率

## 4. QA 模块

### 目标
- 提升答案正确率和证据可信度。

### 迭代项
- 检索召回 + 重排策略优化。
- 回答模板规范（事实与推断分层，证据引用可追溯）。
- badcase 回流和回归测试集沉淀。

### 指标
- QA 正确率
- 证据命中率
- 平均响应时长

## 5. 前端交互与状态一致性

### 目标
- 列表、详情、抽屉状态一致，减少“看起来没生效”。

### 迭代项
- 内化进行中跨页面一致显示。
- 保存后摘要/标签/状态即时回填。
- 表格、滚动条、按钮反馈统一规范。

### 指标
- 状态不同步问题数
- 关键操作完成率
- 用户路径耗时

## 6. 稳定性与可观测

### 目标
- 线上问题可快速定位，改动可快速回归验证。

### 迭代项
- 全链路 trace_id 打通（录入/解析/内化/图谱/QA）。
- 错误码归一和日志结构化。
- benchmark 自动回归与结果沉淀。

### 指标
- 错误率
- MTTR（平均修复时长）
- 回归通过率

## 7. 执行顺序（建议）

1. 录入解析
2. 内化
3. 图谱
4. QA
5. 前端一致性
6. 稳定性与可观测

## 8. 里程碑验收

- M1：录入解析稳定（成功率和时延达标）。
- M2：内化稳定（结构化完整、批量可用）。
- M3：图谱和 QA 达到可用标准。
- M4：全链路指标看板上线并可持续回归。
