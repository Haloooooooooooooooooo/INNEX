## 3. 指标口径（统一）

1. 质量分（1-5）
- 5：稳定、结构完整、几乎无需人工改
- 3：可用但偶有结构/事实问题
- 1：不可用

2. 稳定性
- `success_rate = 成功请求数 / 总请求数`
- 失败分类：`timeout` / `empty` / `format_error` / `other`

3. 性能
- `p95_latency_ms`

4. 成本
- 单次请求成本估算（人民币或美元均可，保持口径一致）

---

## 4. 单任务评分公式

- 质量归一：`quality_norm = quality/5`
- 稳定性归一：`stability_norm = success_rate`
- 延迟归一：`latency_norm = min(1, baseline_p95 / model_p95)`
- 成本归一：`cost_norm = min(1, baseline_cost / model_cost)`

总分：
`score = quality_norm*0.50 + stability_norm*0.25 + latency_norm*0.15 + cost_norm*0.10`

说明：
- `baseline_p95` / `baseline_cost` 使用当前线上默认模型。
- 若模型失败率 > 15%，本轮直接判定不推荐作为默认。
