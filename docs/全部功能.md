# 录入、解析：

录入的时候是否读取全部内容

- 文字：字数少于50个字就不用写了什么都不展示

- 文档：看看文件大小多大，如果小的话可以直接读取，如果文件大的话就要内化才能读取

- 链接是视频：录入的时候不读取原视频，内化的时候才使用手段读取

- 链接是图文笔记（或者就是文字笔记）：直接读取
- 图片：只允许读取10张，ocr
- 只上传附件：如果是一个附件就看看是什么类型的按对应类型处理，如果不止一个附件看看附件总共多大如果加起来都不是很大的话可以直接读取否则一个都不读取



1. 标题（根据不同录入类型的来解析）：

- 文字：有标题直接录入，没有就根据摘要起一个标题，如果很短的画直接取前面5个字。

- 文档：直接用文档名字

- 链接：看看有没有标题，有就直接采用，没有的话看看有没有被读取，如果被读取就通过摘要生成标题，如果没有被读取就显示-

- 图片：根据读取的内容生成，蜀国没有被读取就直接写时间+图片（比如2026/07/06图片）

- 只上传附件：如果是一个附件就看看是什么类型的（文字/文档/图片）按对应类型处理，如果不止一个附件那就直接命名为收录时间+附件组（比如2026/07/06附件组）

  

2. 来源（根据不同类型提取）：

- 文字：文字
- 文档：文档+类型（word、pdf、md、txt看看还有其他类型文档需要读取的吗）
- 链接：优先看是哪个平台的可能可以从链接本身看出来比如小红书b站youtube等等，看不出来就直接写链接
- 图片：就写图片
- 只上传附件：如果是一个附件就看看是什么类型的（文档/图片）按对应类型处理，如果不止一个附件就直接写附件组

3. 收录时间：例如2026-05-14 10:23，按这个格式来

4. 状态：如果录入时候选择稍后看，状态就是稍后看。如果录入时候选择收藏，状态就是待内化

5. 标签：看是否有读取，根据读取的东西（摘要）来打标签，没有读取就写-，等内化的时候再打标签

6. 操作：状态对应按钮

7. 摘要（根据不同录入类型的来解析，如果在这个过程中能够全部读完的要保存这些东西并且产生标签）：

- 文字：字数少于50个字就不用写了什么都不展示，字数大于50字要根据内容生成摘要

- 文档：看是否有读取根据读取的来写摘要，如果没有读取就告诉用户文件过大，内化之后才能生成

- 链接是视频：告诉用户该链接是视频，内化之后才能生成

- 链接是图文笔记：取根据读取的来生成
- 图片：看是否有读取取根据读取的内容来生成，没有读取的话就告诉用户图片太多，内化之后才能读取
- 只上传附件：如果是一个附件就看看是什么类型的按对应类型处理，如果不止一个附件看看有没有读取，没有读取就告诉用户附件太大，内化之后才能读取

8. 我的理解：用户有输入的话直接保存下来，没有的话就展示空

9. 附件：附件清单展示



**1. 录入（Quick Capture）**

1. 用户可录入内容类型：文本、链接（这个链接跳转之后可能是图+文、可能是视频、公众号文章等等）、图片、文档、临时想法。

2. 用户填写“内容输入”主字段（必填）。

3. 用户可填写“我的理解”（选填），用于保留当下判断和上下文。

4. 用户可添加附件（0-N个）：图片、文档、视频

5. 用户选择录入意图：

   稍后看：先存档，后续再处理。

   收藏：进入优先处理队列（待内化）。

6. 用户点击“添加记录”，生成一条新收录项，进入收录箱。

**3. 收录箱（Inbox）**

1. 列表维度：全部、稍后看、待内化、已沉淀。
2. 每条记录展示字段：标题、来源、收录时间、状态、标签、操作。
3. 支持搜索、筛选、按时间查看。
4. 点击记录打开右侧详情抽屉，查看完整信息和可执行动作。

------

**4. 收录状态与操作规则**

1. 稍后看

   可操作：查看原笔记、转待内化、一键内化、删除。

   适用场景：先囤积线索，不立即处理。

2. 待内化

   可操作：查看原笔记、一键内化、删除。

   适用场景：已准备进入结构化沉淀。

3. 已沉淀

   可操作：查看原笔记、基于此笔记提问、知识库定位、删除。

   适用场景：可复用知识资产，进入问答和图谱联动。

------

**5. 详情抽屉（稍后看、待内化）**

1. 展示字段：标题、来源、时间、状态、标签、摘要、我的理解、附件。
2. 加入一个笔记框（用户可以随时在里面做笔记，相当于笔记本），这个目前前端还没有实现需要增加
3. 稍后看底部按钮：查看原笔记、转待内化、一键内化、删除
4. 待内化底部按钮：查看原笔记、一键内化、删除

------

**5. 详情抽屉（已沉淀）**

1. 展示字段：标题、来源、时间、状态、标签、摘要、我的理解、附件。
2. 加入一个笔记框（用户可以随时在里面做笔记，相当于笔记本），这个目前前端还没有实现需要增加
3. AI笔记框，存放内化生成的笔记，点击笔记链接可以跳转到笔记页面（目前前端交互已经实现模拟跳转）
4. 底部按钮：查看原笔记、基于此笔记提问、知识库定位、删除

---

**6. 一键内化（从记录到笔记）**

1. 用户在记录上触发“一键内化”。
2. 系统生成结构化笔记草稿（摘要、关键点、可复用结论、引用依据等）。
3. 用户可在草稿视图检查内容后确认保存。
4. 保存后记录状态更新为已沉淀，并与知识库节点建立关联。
5. 后续可从该笔记继续提问与复用。

------

**7. 操作按钮和详情抽屉底部按钮逻辑**

- 查看原笔记：链接的话跳转到笔记对应网页，文档就直接打开，文字就出现一个弹窗展示（有关闭按钮）
- 转待内化：状态和按钮转变
- 一键内化：
- 删除：会弹toast二次确认
- 基于此笔记提问：已沉淀的状态才可以，弹出右侧凑提和ai助手的对话框并自动挂载该条笔记的全部内容作为上下文（不过上下文可能太大了，后续做到这个功能的时候再讨论）
- 知识库定位：跳转到知识库页面，对应的笔记节点高亮并且展示右抽屉笔记详情页

---

**7. 知识库（图谱页）**：需要找GitHub上面已经实现的知识图谱来构建

1. 结构层级：主题 -> 子主题 -> 笔记节点。
2. 语义层级：笔记和笔记之间
3. 用户可点击节点查看笔记详情页（现在笔记详情页页面被改坏了没有渲染），在已沉淀笔记详情页字段上面加上关联笔记，点击关联笔记之后会跳转成展示该笔记详情页并且知识库对应节点高亮
4. 悬浮在某个点上，和该节点所以相关节点都高亮
5. 从收录箱“知识库定位”可直接跳到对应节点。
6. 知识库用于“看结构”和“找连接”，不是仅看单条内容。

------

**8. AI 助手（基于沉淀知识）**

1. 用户可自由提问，或从“基于此笔记提问”带上下文进入。
2. 助手回答引用已沉淀笔记。把已沉淀的笔记当作知识库
3. 证据足够时展示引用来源，每条来源笔记都可以点击然后在右侧抽屉出现该笔记详情页；证据不足时明确不确定。
4. 用户可把高价值问答“加入笔记/回写知识库”，形成持续沉淀。ai助手的每条回答下面有一个加入笔记按钮，点击之后可以把ai的回答整理进该笔记的详情页（在已沉淀笔记详情页加一栏“ai助手回答”，没有加入过的该栏先不显示出来）

------

**9. 删除与治理**

1. 未沉淀记录删除：移除记录及其附件引用。
2. 已沉淀记录删除：同步移除对应笔记及关联关系（按产品规则执行）。
3. 删除动作为高风险操作，建议二次确认。

------
