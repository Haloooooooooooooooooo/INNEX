# Phase 3 验收文档（内化层升级）

日期：2026-05-28
范围：Phase 3（内化层升级）

## 1. 内化输入规范

内化输入采用三层结构，按优先级拼装：
1. 原始正文主体（primary source）
2. 来源补充内容（supplemental sources）
3. 用户理解引导（user_understanding）

规则：
- `raw_content` 是默认主输入，但不是唯一输入。
- 若来源级主正文不足，允许在内化阶段补齐附件提取文本。
- deferred 场景由内化阶段继续补解析并回写来源（例如远程图片 OCR 来源补齐）。
- 视频解析不是当前前置依赖，仅在 `includeVideo=true` 时作为补充输入。

## 2. 内化正文结构规范

内化正文固定 5 段，顺序固定：
1. `## 核心内容`
2. `## 关键概念 / 关键信息`
3. `## 原文支持要点（事实）`
4. `## 推断与延展（非事实）`
5. `## 可关联方向`

规则：
- 必须显式区分“事实”与“推断”，不可混写。
- 事实段只包含可被原文直接支持的信息。
- 推断段只包含基于事实的延展，不得伪装为事实。
- 若模型输出未满足结构，服务端自动补齐结构骨架。

## 3. 内化正式资产清单

当前已落位的正式资产：
- 正式笔记正文：`notes.content`
- 正式摘要：`notes.summary`
- 正式标签：`notes.tags`
- 正式概念：`notes.concepts`
- 事实层片段（检索向）：`note_chunks`（事实段优先入库）

补充说明：
- 推断层片段当前在正文中分区展示，并以 `[延展片段]` 形式进入补充检索片段。
- capture 层 `summary/tags` 仅为过渡版本，note 层版本为正式版本。

## 4. 内化模型与 Prompt 约束

- capture 与 internalize 的模型策略允许分离。
- internalize prompt 强制“原文依据优先，推断后置”。
- 标签提取与摘要提取保持分任务执行，不混任务。

## 5. 对图谱与 QA 的可用性确认

- 内化结果已具备“事实优先”的可检索结构，支持后续图谱关系判定与 QA 证据引用。
- `note_chunks` 先写入事实片段，再写入内化/延展片段，便于 QA 优先命中可核验证据。
